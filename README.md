# playwright-autopilot

**AI writes your Playwright tests. You run them for free.**

Describe what you want in plain English. Get a real Playwright script — Python or TypeScript — that runs independently with standard tooling. No vendor lock-in. No per-run AI costs.

When selectors break, autopilot heals them and regenerates the script. Back to free.

```
You:       "Search for laptop on Amazon and add the cheapest to cart"
autopilot:  Explores the site, figures out the steps, generates test_amazon.py
pytest:     Runs it. Forever. For free.
```

## The Problem

Playwright MCP burns **114,000 tokens per test run**. Every. Single. Time.

Playwright CLI is free but you write every selector by hand. When the UI changes, you fix them by hand.

Both are wrong. You shouldn't pay AI to click the same button twice, and you shouldn't hand-write selectors AI can figure out.

## How autopilot solves it

```bash
# Step 1: AI explores your site and figures out the steps ($0.03, once)
autopilot explore --url https://amazon.in --goal "search for 1.5 ton AC"

# Step 2: Generate a real Playwright script (free, instant)
autopilot generate playbook.json -o test_amazon.py

# Step 3: Run with standard tooling (free, forever)
pytest test_amazon.py
```

The generated script is **standard Playwright** — no autopilot dependency, works with pytest, trace viewer, CI/CD, everything you already use.

### Generated output (real example from Amazon.in)

```python
# test_amazon.py — generated by autopilot, runs WITHOUT autopilot

import pytest
from playwright.sync_api import Page

VARIABLES = {"query": "1.5 ton split AC"}

def test_amazon_search(page: Page):
    page.goto("https://www.amazon.in")
    page.get_by_role("searchbox", name="Search Amazon.in").fill(VARIABLES["query"])
    page.locator("input[type='submit']").click()
```

That's it. Standard Playwright. Run it a thousand times for free.

### When selectors break

```bash
# Heals broken selectors (free — uses heuristics, not AI)
autopilot replay playbook.json --save

# Regenerate the script
autopilot generate playbook.json -o test_amazon.py
```

10 healing strategies run before AI is ever called. Most breakages are fixed at zero cost.

## Three ways to use it

### 1. CLI (recommended)

```bash
# Explore
autopilot explore --url https://myapp.com --goal "log in and go to settings" --llm claude-sonnet-4-20250514

# Generate
autopilot generate playbook.json --lang python -o test_login.py
autopilot generate playbook.json --lang typescript -o test_login.spec.ts

# Run (no autopilot needed)
pytest test_login.py
npx playwright test test_login.spec.ts
```

### 2. Python library

```python
from autopilot import Agent

async with Agent(llm="claude-sonnet-4-20250514") as agent:
    # AI explores and records a playbook
    result = await agent.explore(
        url="https://myapp.com/login",
        goal="Log in with test@test.com, go to settings, enable dark mode",
        variables={"email": "test@test.com", "password": "secret"},
    )

# Generate standalone Playwright script
from autopilot.codegen import generate_python
code = generate_python(result.playbook, variables={"email": "test@test.com"})
Path("test_login.py").write_text(code)
```

### 3. Interactive REPL

```bash
autopilot interactive --url https://myapp.com --llm claude-sonnet-4-20250514 --headed
```

```
autopilot> fill the email field with test@test.com
  1. fill email (input#email)
autopilot> click sign in
  2. click sign in (button[type='submit'])
autopilot> save login-flow
  Saved: login-flow.json (2 steps)
```

|---|---|---|---|
| **Playwright MCP** | ~$0.15 (114K tokens) | AI writes it every run | No |
| **Playwright CLI** | $0.00 | You write it | No |
| **Browser-Use** | ~$0.10 | AI writes it every run | No |
| **Stagehand** | ~$0.05 | You write it, AI assists | Cache-based |
| **autopilot** | **$0.03 once, then $0.00** | AI writes it once | 10 strategies, zero-token |

## Self-healing (10 strategies, zero AI cost)

When selectors break, autopilot tries 10 heuristic strategies before touching AI:

| Strategy | How |
|---|---|
| Alternative selectors | Fallbacks stored in playbook |
| Label matching | `<label for="...">` association |
| Placeholder matching | `placeholder` attribute |
| ARIA role + name | `get_by_role()` with accessible name |
| Text content | Visible text matching |
| Test ID | `data-testid` attribute |
| Alt text | Image/icon alt text |
| Title attribute | Element title |
| Composite locator | Role + text combined |
| Attribute proximity | Fuzzy match renamed IDs |

All zero tokens. AI (Tiers 2-4) is only called if all 10 fail.

## Also works as MCP server

For AI coding agents (Claude Code, Cursor, Windsurf):

```json
{
  "mcpServers": {
    "autopilot": {
      "command": "autopilot-mcp",
      "args": ["--llm", "claude-sonnet-4-20250514", "--disk-snapshots"]
    }
  }
}
```

7 tools (not 26). `--disk-snapshots` writes page state to files instead of context — the approach Microsoft recommends for coding agents.

## Full CLI

```bash
autopilot explore    # AI explores a site, records playbook
autopilot generate   # Convert playbook to Playwright Python/TypeScript
autopilot replay     # Run playbook with self-healing
autopilot interactive # REPL mode — type commands, watch browser
autopilot import     # Import existing Playwright recordings
autopilot chain      # Run multiple playbooks in sequence
autopilot info       # Show playbook details + health stats
autopilot stats      # Dashboard of all playbooks
```
